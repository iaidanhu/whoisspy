## ADDED Requirements

### Requirement: 排行榜仅限本会话

系统必须仅在当前房间会话内维护排行榜数据；会话结束时必须清空或归档榜单状态，不得被后续新会话直接复用。

#### Scenario: 房间会话结束

- **当** 配置的总局数已完成且房间会话已结束
- **则** 系统清空或归档排行榜数据，确保新会话不沿用旧数据

### Requirement: 按局结算且仅按淘汰顺序加分

系统必须在**每一局游戏结束**时，根据**该局**内玩家被淘汰的先后顺序分配非负分数：第一个在该局被淘汰者 +0，第二个 +1，依此类推；**该局结束时仍存活**的玩家获得该局**最高档**分数；若多名玩家同时存活结束该局（例如卧底胜场剩余 2 人），则这些存活玩家**并列**获得该局最高档分数。

#### Scenario: 一局内多名淘汰后两名存活并列高分

- **当** 该局内已有若干玩家按淘汰顺序出局，且该局以两名存活玩家结束
- **则** 已淘汰者按出局先后依次获得 +0、+1、…；两名存活者**并列**获得该局最高档分数（即若开局 N 人，则最高档为 +(N-1)，两人均得 +(N-1)）

### Requirement: 每局更新累计榜且局间可查看

系统必须在**每一局结束**后更新累计排行榜总分，并允许玩家在本会话任意时刻查看当前累计排名（终局 MVP 揭晓前展示为进行中榜单）。

#### Scenario: 单局结束结算

- **当** 一局游戏结束且该局计分完成
- **则** 系统更新累计总分并对外提供最新排名供局内查看

### Requirement: 最终 MVP 揭晓时机

系统必须在房主配置的**全部局数**完成后才揭晓最终 MVP；在此之前仅展示当前累计排名，不得作为最终 MVP 定稿。

#### Scenario: 终局前查看榜单

- **当** 玩家在最后一局完成前查看排行榜
- **则** 系统仅显示当前累计排名，不进行最终 MVP 揭晓

#### Scenario: 全部局数完成

- **当** 最后一局结束且本会话局数已达房主配置上限
- **则** 系统根据累计最高分公布最终 MVP

### Requirement: 终局排行榜在工作间中部展示

当本会话**全部局数已完成**时，系统必须在 **`/[locale]/r/[code]`** 工作间**主界面中部**展示本会话排行榜（累计分排序及 MVP 信息），布局须符合 `design-ui` 对工作间中部区域的约定；不得以仅模态/抽屉作为唯一展示方式（中部主展示区为必需）。

#### Scenario: 会话打满局数

- **当** 最后一局结算完成
- **则** 工作间中部出现完整排行榜视图，所有在场玩家可见

### Requirement: 房主可重新开始游戏

在终局排行榜展示状态下，**仅房主**可见并可操作 **重新开始游戏**（或等价文案）；点击后系统必须在**同一房间**内开启**新会话**（重置本会话累计分与局进度、回到等待开局状态，房间码不变；具体状态机与 `design.md` 一致）。非房主不得触发该操作。

#### Scenario: 房主点击再来一局

- **当** 终局榜已展示且房主点击重新开始
- **则** 房间进入新会话初始态，榜单数据按新会话规则重置，房主可再次配置（默认值）并开始
