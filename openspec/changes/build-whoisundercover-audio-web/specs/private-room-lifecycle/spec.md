## ADDED Requirements

### Requirement: 私密房间码格式与唯一性

系统必须生成总长度为 8 的房间码：首字符为大写 `W`，后 7 位为随机字母数字字符。系统必须保证该码与当前所有活跃房间码不重复。

#### Scenario: 创建房间返回有效房间码

- **当** 房主创建新房间
- **则** 系统返回长度为 8、以 `W` 开头、且不与任何活跃房间码冲突的房间码

### Requirement: 无登录加入流程

系统必须允许玩家在未登录的情况下，通过房间码、昵称与头像标识加入私密房间。头像展示必须基于 **DiceBear Big Smile** 风格，客户端以 `avatarSeed`（字符串）与固定风格 `bigSmile` 生成头像；服务端保存 `displayName` 与 `avatarSeed`（或等价字段）供全房间同步展示。

#### Scenario: 无账号加入房间

- **当** 玩家提交房间码、昵称（`displayName`）与头像种子（`avatarSeed`）
- **则** 系统将其加入房间并分配会话内玩家身份，并在房间内展示对应昵称与 Big Smile 头像

### Requirement: 契约首页与选择页流程（客户端）

客户端必须将入口拆为两步，且路由须带 **语言段** **`/[locale]/`**（`zh` 或 `en`，见 `app-internationalization` 规格）：**(1)** **`/[locale]/`** 展示契约风身份页（对齐 `home.png`）：用户填写昵称、可选择 Big Smile 头像；若用户**未选择**头像即进入下一步，客户端必须为其分配**随机 `avatarSeed`** 并写入档案。**主操作（如指纹按钮）**进入 **`/[locale]/start`**。**(2)** **`/[locale]/start`** 仅提供 **加入房间** 与 **开房间**，不得与身份页混为同一屏；**视觉必须与契约首页同色纸感**（`design-ui.md` §2.1、§5、`--home-*`）。用户在身份页成功进入下一步时，客户端必须将 `displayName` 与 `avatarSeed` 持久化到 **localStorage**（`whois_profile_v1`）。若用户直接访问 **`/[locale]/start`** 且无有效档案，客户端必须重定向至 **`/[locale]/`**。站点根路径 **`/`** 必须重定向至默认语言（如 **`/zh`**）。

#### Scenario: 再次访问自动预填

- **当** 用户曾保存过本地档案后再次打开 **`/[locale]/`**
- **则** 昵称与头像种子自动预填，用户可修改后再次进入 **`/[locale]/start`**

#### Scenario: 未选手头像则随机默认

- **当** 用户填写合法昵称但未点击任何头像预览即触发「下一步」
- **则** 客户端生成随机 `avatarSeed`（或等价随机字符串）并与昵称一并保存，随后进入 **`/[locale]/start`**

### Requirement: 无准备阶段

系统不得实现独立的「准备」阶段或准备超时逻辑；玩家进入房间后即视为处于可开局状态，由房主控制开始游戏。

#### Scenario: 加入后即可被房主开局

- **当** 房间内玩家数不少于 3 且房主触发开始游戏
- **则** 系统进入发词与对局流程，不要求玩家额外点击准备

### Requirement: 房间人数与开局下限

系统必须将房间内玩家数限制在 3 至 8 人（含）。在玩家数少于 3 时，房主不得开始游戏。

#### Scenario: 房间已满拒绝加入

- **当** 第 9 名玩家尝试加入已达 8 人的房间
- **则** 系统拒绝加入并返回房间已满的提示

#### Scenario: 人数不足禁止开局

- **当** 房主在少于 3 名玩家时尝试开始游戏
- **则** 系统阻止开局并提示最少人数要求

### Requirement: 房主可配置会话局数

系统必须允许房主将**本会话内总局数**配置为 1 至 10（含），表示连续进行多局游戏；不得提供房主自定义词组选项。

#### Scenario: 保存房主局数设置

- **当** 房主确认房间设置
- **则** 系统保存总局数（1–10）作为本会话配置

### Requirement: 仅使用全局系统词库

系统必须从全局系统词库中抽取词对发牌；不得允许房主或玩家提交自定义词组作为当局词语来源。词库数据以项目内 `data/system-word-pairs.json`（或经构建同步的等价数据源）为准。

#### Scenario: 开局发词来源为系统词库

- **当** 一局游戏开始需要分配词语
- **则** 系统仅从系统词库随机选取一对词语，并按规则分配给平民与唯一卧底
