## ADDED Requirements

### Requirement: 每局固定一名卧底

系统必须在每一局游戏中恰好指定 **1** 名卧底角色，且该人数不随房间总人数在 3–8 之间变化而改变。

#### Scenario: 八人局仍仅一名卧底

- **当** 房间内有 8 名玩家且一局游戏开始
- **则** 系统指定其中恰好 1 人为卧底，其余为平民

### Requirement: 胜负条件与单局结束

系统必须在满足以下任一条件时**立即结束当前局**并公布该局结果：**(A)** 卧底在投票流程中被淘汰出局，平民胜；**(B)** 场上存活玩家仅剩 2 人且其中恰好 1 人为卧底，卧底胜。

#### Scenario: 卧底被淘汰平民胜

- **当** 投票结果导致卧底玩家出局
- **则** 系统结束该局，判定平民方获胜

#### Scenario: 剩两人且含卧底则卧底胜

- **当** 投票与淘汰流程结束后场上存活玩家为 2 人，且卧底仍在其中
- **则** 系统结束该局，判定卧底胜

### Requirement: 发言阶段与描述消息权限一致

在发言阶段，系统必须将「当前发言人」与文字描述通道的发送权限绑定：**仅当前轮到发言的玩家**可发送本轮描述文本；该约束与 `room-text-channel` 规格一致，并由服务端在发言阶段内强制执行。

#### Scenario: 阶段与发言人字段一致

- **当** 状态机将发言指针切换到某一座位玩家
- **则** 系统仅将该玩家视为有权发送描述消息的玩家，直至该指针切换

### Requirement: 按加入顺序固定发言顺序

系统必须按玩家加入顺序分配座位序号，并在发言阶段按座位序号升序（界面从左到右）依次发言。

#### Scenario: 回合进入发言阶段

- **当** 某轮发言阶段开始
- **则** 系统按座位序号从小到大确定发言顺序

### Requirement: 发言超时自动推进

系统必须在当前发言者计时结束时自动结束其回合；若仍有未发言者则切换至下一位；若无下一位则进入投票阶段。

#### Scenario: 当前发言者超时

- **当** 当前发言者剩余时间为零
- **则** 系统立即结束该回合并切换至下一有效发言轮次或进入投票阶段

### Requirement: 投票不可投自己且可弃票

在投票阶段，系统必须拒绝玩家将票投给自己的请求；玩家必须能够主动选择弃票。

#### Scenario: 投自己无效

- **当** 玩家提交的目标为自己座位
- **则** 系统拒绝该票并提示不可投自己

#### Scenario: 主动弃票

- **当** 玩家在计时内选择弃票
- **则** 系统将该玩家记为本轮弃票

### Requirement: 投票阶段时长八秒与超时弃票

投票阶段倒计时长度必须为 **8 秒**。系统必须在计时结束时，对仍未提交有效投票（未投他人）的玩家自动记为**弃票**。

#### Scenario: 超时未投票

- **当** 投票阶段 8 秒结束且某玩家未提交有效票
- **则** 系统将该玩家视为本轮弃票

### Requirement: 全员弃票则无人出局

若本轮所有存活玩家的投票结果均为弃票（没有任何一票投给其他玩家），系统必须判定**本轮无人出局**，并进入下一轮发言/投票流程（该局未因本条件结束）。

#### Scenario: 全部弃票

- **当** 投票结算时每一名存活玩家均为弃票
- **则** 系统不淘汰任何玩家，并推进至下一轮游戏流程

### Requirement: 平票加赛与投票范围限制

当存在得票且**最高得票**为平局（多名玩家并列最高票）时，系统必须进入加赛：仅平票玩家按座位顺序重新发言，且本轮投票的有效目标**仅限**平票玩家集合（仍不可投自己、可弃票、8 秒超时规则不变）。

#### Scenario: 多名玩家并列最高票

- **当** 计票后有两名或以上存活玩家并列最高得票数（且非全员弃票情形）
- **则** 系统进入平票加赛，发言与可投票目标限制在平票玩家集合内

### Requirement: 平票加赛无次数上限

系统必须在没有配置最大迭代次数的前提下持续平票加赛循环，直到产生**非平票的最高票出局**或回到「可继续」状态（例如加赛路径上规则允许的等价结果）。

#### Scenario: 加赛后仍平票

- **当** 一次平票加赛投票结束后再次出现最高票平局
- **则** 系统再次进入新一轮平票加赛
