# UI 设计规范（MVP）

本文档用于**约束**实现时的视觉与布局，减少「每次生成都不一样」的随机性。  
**规则**：新增页面类型、新主色、新布局结构前，须先改本文档再写代码。

**国际化**：绝大多数面向用户的可见文案须通过 **next-intl** 从 **`messages/zh.json` / `messages/en.json`** 读取；**例外**：**Logo 字标固定为 `Who Is Spy`**（品牌字标，**不翻译、不随 locale 切换**，见 [`docs/project-meta.md`](../../../docs/project-meta.md)）。其余文案实现时须 **中英各写一条** 键值。详见 `design.md` 与 `app-internationalization` 规格。

## 1. 产品气质与参考方向

- **定位**：线下派对游戏的线上替代；**偏 C 端**、有剧场感与仪式感。
- **首页关键词**：羊皮纸、酒红点缀、衬线大标题、签名线、指纹印、克制装饰。
- **对局关键词**：深暖沉浸、**金/琥珀**高光、玻璃拟态、**左右双列玩家**、中部「舞台」、倒计时**强可见**。
- **禁止**：
  - 冷灰蓝「企业后台」主色、大面积浅灰表格风。
  - 高饱和荧光绿/红铺满背景。
  - 首页与对局页共用同一套背景色令牌不加区分。
- **组件与图标**：交互原语以 **Radix UI** 为准（无默认皮肤，样式用 Tailwind 4 + 本文令牌）；图标统一 **Lucide React**（如指纹按钮用 `Fingerprint`）。客户端跨组件状态用 **Jotai**；过渡与布局动画用 **Framer Motion**，须符合 §7.4 / §10 的克制要求。

## 2. 本地身份档案与两步入口流程

### 2.1 路由与步骤

1. **`/[locale]/`（契约首页）**，`locale` 为 `zh` | `en`  
   - 设置 **昵称**（签名式输入）与 **Big Smile 头像**（可选）。  
   - 主操作：**圆形指纹按钮** = 「下一步」，进入步骤 2（**不**在本页直接弹房间码输入）。  
2. **`/[locale]/start`（选择页）**  
   - 两个入口：**加入房间**（输入房间码）、**开房间**（创建）。  
   - **视觉强制与契约首页同色纸感**：全页背景、主容器须使用与 §5 相同的 **`--home-*` 令牌**（米色纹理底、羊皮纸卡片、酒红点缀、双线框或可识别的同系纸卡变体），**禁止**对局页深色全屏、**禁止**冷灰/纯白极简替代纸感；仅允许在卡片内部排版上与首页区分（如双按钮区、房间码输入条）。

从 **`/[locale]/start`** 再进入 **`/[locale]/r/[code]`** 或创建后跳转房间。站点根 `/` **重定向**至默认语言（如 `/zh`），见 `design.md` 国际化小节。

### 2.2 昵称与头像规则

- **昵称**：非空方可点「下一步」；长度建议 2～16；占位与辅助文案对齐参考图语义（如「签署你的名字」「签署名字后才可生效」可本地化微调，**须保留横线 + 下划线输入**气质）。
- **头像**：展示 **DiceBear Big Smile** 可选列表（横向滚动或紧凑网格，**圆形**外框，与指纹按钮同为圆形态语言）。  
- **默认头像**：若用户**从未点击过**任一头像选项，则在点击「下一步」时（或进入首页首次渲染时，二选一但须整站一致）自动采用 **随机 `avatarSeed`**（如 `crypto.randomUUID()`）写入档案，保证每人必有稳定头像；**用户点选后**以点选 seed 为准。

### 2.3 localStorage

- 在 **「下一步」成功**时（或昵称与 seed 有效时即时）写入 **`localStorage`**，键名建议不变：

| 键 | 类型 | 说明 |
|----|------|------|
| `whois_profile_v1` | JSON 字符串 | `{ "displayName": string, "avatarSeed": string }` |

- **再次访问**：打开 **`/[locale]/`** 时预填昵称与头像；用户可改后再次「下一步」进入 **`/[locale]/start`**（`locale` 以当前 URL 为准）。

### 2.4 与房间的关系

- **创建 / 加入房间**请求必须携带当前档案中的 `displayName` + `avatarSeed`。
- 房间内**不强制**提供改头像入口（MVP）；改资料返回 **`/[locale]/`**（保持当前语言）。

## 3. 头像：DiceBear Big Smile（npm）

### 3.1 依赖与文档

- 使用 **npm** 接入：**`@dicebear/core`** + **`@dicebear/collection`**，风格为 **`bigSmile`**（对应官方 **Big Smile**）。
- 官方说明与选项见：[Big Smile – DiceBear](https://www.dicebear.com/styles/big-smile/)。
- **许可**：DiceBear 代码 MIT；**Big Smile 造型设计为 CC BY 4.0**，产品关于页或设置页保留简短署名/链接（见 DiceBear 页面 **LICENSE** 区块）。

### 3.2 实现要点

- **禁止**在 MVP 用 HTTP API 拉 SVG 作为主路径（避免多余网络与 CSP 问题）；**优先**在客户端用 `createAvatar(bigSmile, { seed, ... })` 生成 SVG 字符串或 Data URL，与 Next/Image 策略协调（可用 `dangerouslySetInnerHTML` 封装组件或转 data URL）。
- 用户在 **`/[locale]/`** 通过 **横向滚动或紧凑网格** 预选 `seed`（预设列表 + 可选「换一批」随机 seed）；**首页**预览可略大于对局列表（如 56～72px），对局内仍 **40～48px**。

## 4. 信息架构与页面清单

以下 **6 类路由级页面**为 MVP 全部；不得在未补充文档的情况下增加第 7 类「主流程页」。

| 路由（建议） | 名称 | 核心内容 |
|--------------|------|----------|
| `/` | **根** | **重定向**至 `/zh`（或默认 `locale`） |
| `/[locale]/` | **契约首页** | 对齐 [`home.png`](../../../docs/images/home.png)：纸卡片、标题、宣誓文案区、**横线签名昵称**、**Big Smile 头像选择**、**指纹按钮 → 下一步**；**文案走 i18n**；localStorage |
| `/[locale]/start` | **选择页** | **加入房间**、**开房间**；**同色纸感**（`--home-*`）；**语言切换**；无档案则重定向 **`/[locale]/`** |
| `/[locale]/r/[code]` | 房间内（未开局） | 对齐 [`image.png`](../../../docs/images/image.png)：顶栏 + 左右玩家列、房间码、房主局数与开始 |
| `/[locale]/r/[code]/play` | 对局中 | 同参考图：顶栏、双列玩家、中部舞台、底栏、FAB |
| 同上（模态/抽屉） | 排行榜 | 玻璃深色系，与对局一致 |
| 同上（全屏层） | MVP 揭晓 | 金色强调冠军卡 |

**导航原则**：对局尽量少跳路由；`play` 单页 + 覆盖层即可。**`/ → /[locale]/ → /[locale]/start → 房间`** 为固定心智顺序。

## 5. 首页（契约风）布局与令牌 — 对齐 `home.png`

### 5.1 结构

- **全屏背景**：柔和**纹理米色/浅灰**（非纯白），`--home-bg-page`。
- **居中主卡片**：竖向矩形，**双线细边框**（`double` 或内外 `border` 模拟），`--home-paper`，**轻阴影**托起纸感。
- **标题区**：主标题字标固定为 **`Who Is Spy`**（大写衬线、`font-serif`、`tracking-[0.2em]`，颜色 **`--home-ink-burgundy`**，**不随 locale 翻译**）；其下副标语（如类似 *THE SHADOW GAME* 气质）走 **i18n**；再下为中文/英文宣誓说明文案（i18n）。
- **正文**：2～3 行居中中文衬线感说明（可用系统衬线或 `font-serif`）。
- **昵称区**：标签「签署你的名字」+ **无框输入**，文字落在**酒红横线**上（`border-b-2`），占位「在此签名…」类文案。
- **头像区**：标签「选择头像」或融入宣誓文案下方；**横向滚动**小圆头像，选中态 **酒红细环**（勿用对局页金色粗环）。
- **主 CTA**：卡片底部居中 **大圆按钮**，深灰底、内凹立体感、**指纹图标**（Lucide `Fingerprint`）；点击跳转 **`/[locale]/start`**（昵称合法前提下）。
- **角标装饰**：卡片内下角小图标（冠/盾形简化），酒红色，与参考一致。

### 5.2 首页专用 CSS 变量（与 §7 对局令牌并存）

| 令牌 | 用途 | 建议值 |
|------|------|--------|
| `--home-bg-page` | 全页背景 | `#e8e4dc` 或带噪点纹理 |
| `--home-paper` | 卡片底 | `#f5f1e6` |
| `--home-ink-burgundy` | 标题、分割线、选中环 | `#8b2323` |
| `--home-text-muted` | 副文案 | `#6b6560` |
| `--home-seal-btn` | 指纹按钮填充 | `#3d3a36` |
| `--home-seal-btn-inner` | 指纹图标色 | `#9a9590` |

## 6. 对局页全局布局结构（对齐 `image.png`）

### 6.1 桌面（`lg+`）

```
┌─────────────────────────────────────────────────────────────┐
│ 顶栏：Logo | 局进度/存活数/阶段 Pill | 本局身份(仅本人可见?) │ 设置 │
├──────┬──────────────────────────────────────────────┬──────┤
│ 左列 │              中部「舞台」区域                    │ 右列 │
│ 玩家 │   上方：浮动系统/描述气泡（时间线）                │ 玩家 │
│ 卡片 │   中/下：可选装饰插画、大倒计时                   │ 卡片 │
│ 列表 │   底部：宽主提示条（当前阶段文案）                │ 列表 │
├──────┴──────────────────────────────────────────────┴──────┤
│                                    [FAB 输入/发言] ← 右下     │
└─────────────────────────────────────────────────────────────┘
```

- **玩家分配**：3～8 人时 **尽量均分**左右列（如左 4 右 4、左 3 右 2）；**当前用户**卡片 **金色描边 + `YOU` 角标**（与参考图一致）。
- **投票**：点击对方玩家卡片即投票（或打开投票模式高亮可选卡片）；自己卡片不可点、置灰。

### 6.2 移动端（`&lt;lg`）

- 顶栏保留（压缩文案）。
- 玩家列表改为 **横向滚动条**置于舞台**上方**或 **Tab「玩家 | 游戏」**；中部与 FAB **保留**（FAB 贴底安全区）。
- **禁止**在窄屏强行三列挤压导致不可点。

## 7. 设计令牌 — 对局主题（`image.png`）

实现时写入 **Tailwind `theme.extend` + CSS 变量**（`globals.css` `:root`），组件**只引用令牌名**，禁止散落 `#hex`。对局相关类名可加 `data-theme="game"` 或根选择器 `.theme-game`。

### 7.1 颜色（暖深底 + 金强调）

| 令牌名 | 用途 | 建议值 |
|--------|------|--------|
| `--bg-base` | 页背景 | `#1a1010`（深暖炭灰/勃艮第黑） |
| `--bg-elevated` | 顶栏、主面板 | `rgba(0,0,0,0.35)` + `backdrop-blur-md` |
| `--bg-card` | 玩家卡片、气泡 | `rgba(255,255,255,0.05)` + `backdrop-blur-sm` |
| `--border-subtle` | 默认描边 | `rgba(255,255,255,0.10)` |
| `--text-primary` | 主文案 | `#f5f0e8`（偏奶油白） |
| `--text-secondary` | 次要 | `rgba(245,240,232,0.55)` |
| `--accent` | 主强调、主按钮、当前阶段 | `#d4a853`（金） |
| `--accent-bright` | 标题高亮、hover | `#e8c97c` |
| `--accent-dim` | 暗金背景 | `rgba(212,168,83,0.12)` |
| `--danger` | 出局、危险操作 | `#c45c5c` |
| `--success` | 平民胜等 | `#6d9f7b` |
| `--warning` | 倒计时警戒 | `#e8b86d` |

**发光**（仅小面积：当前发言人描边、FAB、主按钮）：

| 令牌 | 值 |
|------|-----|
| `--glow-accent` | `0 0 20px rgba(212,168,83,0.35)` |

### 7.2 圆角与描边

- 玩家卡片、主面板：`rounded-xl`～`rounded-2xl`
- 按钮、输入：`rounded-xl`
- 头像：`rounded-full`
- 主提示底条：宽条 `rounded-2xl`，`border border-white/10`

### 7.3 字体

- 中文栈：`"PingFang SC","Microsoft YaHei",ui-sans-serif,system-ui,sans-serif`
- **阶段标题 / 顶栏关键数字**：可用 `tracking-wide`，颜色 `--accent` 或 `--accent-bright`
- 正文仍以 `--text-primary` 为主，避免全屏金色字

### 7.4 动效

- 阶段切换、`transition-all duration-200`
- 投票倒计时最后 3 秒：数字与环用 `--warning`，禁止全屏闪动

## 8. 核心组件规格 — 对局页（在 `image.png` 语义下）

### 8.1 顶栏

- 左：**产品 Logo + 字标**：字标文案**固定为** **`Who Is Spy`**（与 [`docs/project-meta.md`](../../../docs/project-meta.md) 一致；**不参与 i18n**，`zh` / `en` 路由下均同形）；左侧可配小图标（狼/剪影等）与参考图布局一致即可。
- 中：**局数**（如「第 2 / 5 局」）、**存活**（如 `存活 6/8`）、**阶段 Pill**（玻璃胶囊：`描述中` / `投票中` / `平票加赛` 等）。
- 右：**本局身份**（仅本人可见：平民 / 卧底，文案勿剧透他人）+ **设置**（音效开关等）。

### 8.2 玩家卡片

- 左头像（DiceBear 圆图）+ 编号 + 昵称 + 可选一行状态（「已出局」）。
- **自己**：`border-2 border-[var(--accent)]` + `YOU` 徽章。
- **当前发言人**：在金色基础上可加 `--glow-accent`。
- **已出局**：`opacity-45`，不可交互。

### 8.3 中部舞台

- **浮动气泡**：系统消息、他人描述摘要（短句）；深底半透明，**小圆角**。
- **底部主提示条**：宽、半透明，写清「当前轮到谁 / 请投票 / 平票加赛说明」等，对应参考图底部大对话框。

### 8.4 FAB（右下角）

- 圆形，`border-2 border-[var(--accent)]`，内图标 **铅笔/编辑**（Lucide `Pencil` 或 `MessageSquare`）。
- **行为**：发言阶段打开输入面板；投票阶段可改为「打开投票」或直接在中部展示投票（二选一须整站一致，**推荐投票阶段中部常驻投票区，FAB 仅发言**）。

### 8.5 按钮

- **主按钮**：金底深字或深底金边（保证对比度 ≥ WCAG AA）；hover 用 `--accent-bright`。
- **次按钮**：玻璃 + `border-white/15`。

### 8.6 描述输入 / 投票

- 输入区打开时可从 FAB 以 **底部抽屉** 滑出（H5），PC 可用右侧抽屉或居中模态，风格统一玻璃面板。
- 投票 **8 秒**环形进度：金 → 琥珀警戒色。

### 8.7 模态 / 抽屉

- 背景：`bg-black/60 backdrop-blur-sm`
- 容器：`bg-[var(--bg-card)] border border-white/10 rounded-2xl`

## 9. 响应式断点

- **`lg`（1024px）及以上**：启用 **左右双列玩家 + 中部舞台** 主布局。
- **以下**：折叠为 **顶栏 + 玩家横滑/Tab + 舞台 + FAB**，逻辑不变。
