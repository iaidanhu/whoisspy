# UI 设计规范（MVP）

本文档用于**约束**实现时的视觉与布局，减少「每次生成都不一样」的随机性。  
**规则**：新增页面类型、新主色、新布局结构前，须先改本文档再写代码。

**国际化**：绝大多数面向用户的可见文案须通过 **next-intl** 从 **`messages/zh.json` / `messages/en.json`** 读取；**例外**：**Logo 字标固定为 `Who Is Spy`**（品牌字标，**不翻译、不随 locale 切换**，见 [`docs/project-meta.md`](../../../docs/project-meta.md)）。其余文案实现时须 **中英各写一条** 键值。详见 `design.md` 与 `app-internationalization` 规格。

## 1. 产品气质与参考方向

- **定位**：线下派对游戏的线上替代；**偏 C 端**、有剧场感与仪式感。
- **首页关键词**：羊皮纸、酒红点缀、衬线大标题、签名线、指纹印、克制装饰。
- **对局关键词**：深暖沉浸、**金/琥珀**高光、玻璃拟态、**左右双列玩家**、中部「舞台」、倒计时**强可见**。
- **禁止**：
  - 冷灰蓝「企业后台」主色、大面积浅灰表格风。
  - 高饱和荧光绿/红铺满背景。
  - 首页与对局页共用同一套背景色令牌不加区分。
- **组件与图标**：**基础组件优先 [shadcn/ui](https://ui.shadcn.com/)**（Radix 之上、Tailwind 可主题化，与本文令牌对齐后使用 **Button、Textarea、Dialog、Dropdown** 等）；未覆盖的交互原语直接用 **Radix**。图标统一 **Lucide React**（如指纹按钮用 `Fingerprint`）。客户端跨组件状态用 **Jotai**；过渡与布局动画用 **Framer Motion**，须符合 §7.4 / §10 的克制要求。

## 2. 本地身份档案与两步入口流程

### 2.1 路由与步骤（首页双态）

**`/[locale]/`（根语言首页）** 行为由 **`whoisspy_profile_v1`** 是否存在且有效决定：

- **无档案（首次或未登记）**：展示 **契约登记**（对齐 `home.png`）— 昵称、Big Smile 头像、**圆形指纹按钮** = 「下一步」；成功后写入 **`whoisspy_profile_v1`** 并进入 **`/[locale]/start`**（**不**在本页直接弹房间码输入）。
- **已有档案**：**不得**将契约登记作为首屏；应 **重定向至** **`/[locale]/start`**（或客户端等价：直接渲染选择页）。用户改资料可走 **`/[locale]/?edit=1`** 等约定入口再显示契约表单。

**`/[locale]/start`（选择页）**

- **加入房间**：输入房间码 → **确定** → **`/[locale]/r/[code]`**。
- **开房间**：创建成功后 **直接进入** **`/[locale]/r/[code]`** 工作间（与对局同一深暖壳；创建者 = 房主 = **1 号位**）。
- **视觉强制与契约首页同色纸感**：全页背景、主容器须使用与 §5 相同的 **`--home-*` 令牌**，**禁止**对局页深色全屏替代纸感。

无档案访问 **`/start`** 时须重定向 **`/[locale]/`**。站点根 `/` **重定向**至默认语言（如 `/zh`），见 `design.md`。

### 2.2 昵称与头像规则

- **昵称**：非空方可点「下一步」；长度建议 2～16；占位与辅助文案对齐参考图语义（如「签署你的名字」「签署名字后才可生效」可本地化微调，**须保留横线 + 下划线输入**气质）。
- **头像**：展示 **DiceBear Big Smile** 可选列表（横向滚动或紧凑网格，**圆形**外框，与指纹按钮同为圆形态语言）。  
- **默认头像**：若用户**从未点击过**任一头像选项，则在点击「下一步」时（或进入首页首次渲染时，二选一但须整站一致）自动采用 **随机 `avatarSeed`**（如 `crypto.randomUUID()`）写入档案，保证每人必有稳定头像；**用户点选后**以点选 seed 为准。

### 2.3 localStorage

- 在 **「下一步」成功**时（或昵称与 seed 有效时即时）写入 **`localStorage`**：

| 键 | 类型 | 说明 |
|----|------|------|
| `whoisspy_profile_v1` | JSON 字符串 | `{ "displayName": string, "avatarSeed": string }` |
| `whoisspy_locale_v1` | 字符串 | `zh` \| `en`，语言偏好（与 `app-internationalization` 规格一致） |

- **再次访问**：有档案时 **`/[locale]/`** 应直达 **`/start`**；在 **编辑资料** 模式下打开契约页时预填昵称与头像，用户可改后再次「下一步」进入 **`/[locale]/start`**。

### 2.4 与房间的关系

- **创建 / 加入房间**请求必须携带当前档案中的 `displayName` + `avatarSeed`。
- 房间内**不强制**提供改头像入口（MVP）；改资料返回 **`/[locale]/`**（保持当前语言）。

## 3. 头像：DiceBear Big Smile（npm）

### 3.1 依赖与文档

- 使用 **npm** 接入：**`@dicebear/core`** + **`@dicebear/collection`**，风格为 **`bigSmile`**（对应官方 **Big Smile**）。
- 官方说明与选项见：[Big Smile – DiceBear](https://www.dicebear.com/styles/big-smile/)。
- **许可**：DiceBear 代码 MIT；**Big Smile 造型设计为 CC BY 4.0**，产品关于页或设置页保留简短署名/链接（见 DiceBear 页面 **LICENSE** 区块）。

### 3.2 实现要点

- **禁止**在 MVP 用 HTTP API 拉 SVG 作为主路径（避免多余网络与 CSP 问题）；**优先**在客户端用 `createAvatar(bigSmile, { seed, ... })` 生成 SVG 字符串或 Data URL，与 Next/Image 策略协调（可用 `dangerouslySetInnerHTML` 封装组件或转 data URL）。
- 用户在 **`/[locale]/`** 通过 **横向滚动或紧凑网格** 预选 `seed`（预设列表 + 可选「换一批」随机 seed）；**首页**预览可略大于对局列表（如 56～72px），对局内仍 **40～48px**。

## 4. 信息架构与页面清单

MVP **主流程路由**如下；**工作间**为 **单一路径** **`/[locale]/r/[code]`**，用 **房间状态** 切换等待成员、对局中、**终局中部排行榜**；**不强制**独立 `/play` 子路由（若实现拆分，须同一视觉壳与实时上下文）。

| 路由（建议） | 名称 | 核心内容 |
|--------------|------|----------|
| `/` | **根** | **重定向**至 `/zh`（或默认 `locale`） |
| `/[locale]/` | **语言首页** | **无档案**：契约登记（[`home.png`](../../../docs/images/home.png)）。**有档案**：重定向 **`/start`** |
| `/[locale]/start` | **选择页** | **加入**（房间码 + 确定）、**开房**（直达工作间）；**同色纸感**（`--home-*`）；**语言切换**；无档案 → 重定向 **`/[locale]/`** |
| `/[locale]/r/[code]` | **工作间（单壳）** | 对齐 [`image.png`](../../../docs/images/image.png)：顶栏、双列玩家、中部舞台、底栏、FAB；含 **等待** / **对局** / **终局榜（中部主区）** |
| 工作间内 | **游戏设置（房主）** | 抽屉/面板：投票时长、发言时长、总局数等，**均有默认值** |
| 工作间内 | **终局** | **中部**展示累计榜与 MVP；**仅房主**可见 **重新开始游戏** |

**导航原则**：**`/ → /[locale]/（登记或跳过）→ /[locale]/start → /r/[code]`**；对局尽量少跳路由。

## 5. 首页（契约风）布局与令牌 — 对齐 `home.png`

### 5.1 结构

- **全屏背景**：柔和**纹理米色/浅灰**（非纯白），`--home-bg-page`。
- **居中主卡片**：竖向矩形，**双线细边框**（`double` 或内外 `border` 模拟），`--home-paper`，**轻阴影**托起纸感。
- **标题区**：主标题字标固定为 **`Who Is Spy`**（大写衬线、`font-serif`、`tracking-[0.2em]`，颜色 **`--home-ink-burgundy`**，**不随 locale 翻译**）；其下副标语（如类似 *THE SHADOW GAME* 气质）走 **i18n**；再下为中文/英文宣誓说明文案（i18n）。
- **正文**：2～3 行居中中文衬线感说明（可用系统衬线或 `font-serif`）。
- **昵称区**：标签「签署你的名字」+ **无框输入**，文字落在**酒红横线**上（`border-b-2`），占位「在此签名…」类文案。
- **头像区**：标签「选择头像」或融入宣誓文案下方；**横向滚动**小圆头像，选中态 **酒红细环**（勿用对局页金色粗环）。
- **主 CTA**：卡片底部居中 **大圆按钮**，深灰底、内凹立体感、**指纹图标**（Lucide `Fingerprint`）；点击跳转 **`/[locale]/start`**（昵称合法前提下）。
- **角标装饰**：卡片内下角小图标（冠/盾形简化），酒红色，与参考一致。

### 5.2 首页专用 CSS 变量（与 §7 对局令牌并存）

| 令牌 | 用途 | 建议值 |
|------|------|--------|
| `--home-bg-page` | 全页背景 | `#e8e4dc` 或带噪点纹理 |
| `--home-paper` | 卡片底 | `#f5f1e6` |
| `--home-ink-burgundy` | 标题、分割线、选中环 | `#8b2323` |
| `--home-text-muted` | 副文案 | `#6b6560` |
| `--home-seal-btn` | 指纹按钮填充 | `#3d3a36` |
| `--home-seal-btn-inner` | 指纹图标色 | `#9a9590` |

## 6. 对局页全局布局结构（对齐 `image.png`）

### 6.1 桌面（`lg+`）

```
┌─────────────────────────────────────────────────────────────┐
│ 顶栏：Logo | 局进度/存活数/阶段 Pill | 本局身份(仅本人可见?) │ 设置 │
├──────┬──────────────────────────────────────────────┬──────┤
│ 左列 │              中部「舞台」区域                    │ 右列 │
│ 玩家 │   上方：浮动系统/描述气泡（时间线）                │ 玩家 │
│ 卡片 │   中/下：可选装饰插画、大倒计时                   │ 卡片 │
│ 列表 │   底部：宽主提示条（当前阶段文案）                │ 列表 │
├──────┴──────────────────────────────────────────────┴──────┤
│                                    [FAB 输入/发言] ← 右下     │
└─────────────────────────────────────────────────────────────┘
```

- **玩家分配**：3～8 人时 **尽量均分**左右列（如左 4 右 4、左 3 右 2）；**当前用户**卡片 **金色描边 + `YOU` 角标**（与参考图一致）。
- **投票**：点击对方玩家卡片即投票（或打开投票模式高亮可选卡片）；自己卡片不可点、置灰。

### 6.2 移动端（`&lt;lg`）

- 顶栏保留（压缩文案）。
- 玩家列表改为 **横向滚动条**置于舞台**上方**或 **Tab「玩家 | 游戏」**；中部与 FAB **保留**（FAB 贴底安全区）。
- **禁止**在窄屏强行三列挤压导致不可点。

## 8. 核心组件规格 — 对局页（在 `image.png` 语义下）

### 8.1 顶栏

- 左：**产品 Logo + 字标**：字标文案**固定为** **`Who Is Spy`**（与 [`docs/project-meta.md`](../../../docs/project-meta.md) 一致；**不参与 i18n**，`zh` / `en` 路由下均同形）；左侧可配小图标（狼/剪影等）与参考图布局一致即可。
- 中：**局数**（如「第 2 / 5 局」）、**存活**（如 `存活 6/8`）、**阶段 Pill**（玻璃胶囊：`描述中` / `投票中` / `平票加赛` 等）。
- 右：**本局身份**（仅本人可见：平民 / 卧底，文案勿剧透他人）+ **声音设置**（全玩家可有）：至少支持 **静音/总开关**；若有分项，建议 **BGM**（等待开局循环曲，见 `design.md` §7）与 **短音效**（投票、发言提示、预警、开局/局终等）可区分或合并为一键，文案走 i18n。
- **房主专属（与上并列或紧邻，仅房主可见）**：**游戏设置**（打开面板改投票时长、发言时长、总局数等；**有默认值，可不点开即开局**）+ **开始游戏**（未满 3 人等业务条件不满足时为 **disabled**）。**非房主不得出现**上述两项。

### 8.2 玩家卡片（侧栏，对齐 [`image.png`](../../../docs/images/image.png)）

- 结构：**圆头像**（DiceBear）+ **座位号** + **昵称** + 可选副行（出局、状态文案）；卡片圆角矩形、玻璃/深底与参考图一致。
- **自己（任意阶段）**：`border-2 border-[var(--accent)]` + **`YOU` /「你」徽章**（i18n）；与参考图「你 + 身份」同层级信息架构（卧底/平民仅本人可见，勿剧透他人）。
- **当前发言人（发言阶段）**：须在侧栏 **一眼可辨**，效果 **强于** 非发言人：
  - **金色描边 + 外发光**（`--accent` + `--glow-accent`），卡片背景可 **略提亮** 相对邻卡；
  - 可与「自己」态 **叠加**（本人且正在发言 = 高亮 + YOU）。
- **已出局**：`opacity-45`，不可交互；不参与「当前发言人」高亮。
- **投票等其它阶段**：当前发言人高亮规则可按阶段收缩为「轮到操作的人」或仅中部提示，但 **发言阶段** 必须遵守本条。

### 8.3 中部舞台 — 发言阶段（主视觉，对齐 [`image.png`](../../../docs/images/image.png)）

中部区域是 **剧场主舞台**，与左右侧栏 **联动**：谁发言，舞台就 **以谁为中心**。

**A）当前发言人不是本机（观战）**

- **上方**：可选 **半透明气泡流** — 系统/历史短句（如阶段切换、上一位摘要），小圆角，不抢主视觉。
- **主体**：
  - **大号头像**：当前发言人的 DiceBear 头像，尺寸 **明显大于侧栏内小头像**（建议 PC 约 **140～200px** 直径，H5 等比缩小但仍为舞台中心）。
  - **主文案带**：舞台 **偏下** 的 **宽幅半透明条**（对应参考图底部大对话框气质）：展示该玩家 **已发送的本轮描述全文**；若其尚未发送，显示 **占位提示**（如「请 N 号发言…」，i18n），可配 **发言倒计时** 环/数字。

**B）当前发言人是本机（输入态）**

- 舞台 **同一区域切换为输入态**（**不再**依赖 FAB 作为唯一入口；见 §8.4）：
  - 保留 **本人大号头像** + 简短提示（如「轮到你描述」）。
  - **主输入**：使用 **多行 Textarea**（实现上优先 **shadcn/ui `Textarea`**，底层即原生 `<textarea>`，**默认支持 Shift+Enter 插入换行**）。占据主文案带区域，**玻璃深底、金/琥珀描边或底边**，占位符走 i18n；最小高度与最大高度须适配 H5（可 `max-h` + 内滚）。
  - **键盘约定**（须在占位旁或帮助文案 **i18n** 简短说明）：**Enter** → **发送**（对 `keydown` 拦截默认换行）；**Shift+Enter** → **换行**（保持 textarea 默认行为）。空内容或仅空白时发送须 **禁止** 或 **trim 后为空则禁止**。
  - **语音入口**：Textarea 旁 **麦克风图标按钮**（Lucide `Mic`）；点击触发 **浏览器语音转文字**（Web Speech API），识别结果 **追加或填入** 当前 Textarea，用户可编辑后再发；不可用则隐藏或禁用并走键盘-only（见 `room-text-channel`）。
  - **发送成功**：清空输入、退出本机输入态；由服务端推进 **下一位发言人或下一阶段**（**指结束本轮本人发言**，**不是**离开房间 / 关闭 WebSocket）。
- Textarea 须有 **`aria-label`** / 关联 label；发送中 **禁用重复提交**；可选 **显式「发送」按钮**作辅助（与 Enter 等价），须无障碍可达。

**C）其它阶段（投票、平票加赛等）**

- 中部改展示 **投票面板**、**阶段说明** 等，侧栏高亮规则见各阶段补充；**发言阶段** 以本条为准。

### 8.4 FAB（右下角）

- 视觉：圆形，`border-2 border-[var(--accent)]`，内图标 **铅笔/编辑**（Lucide `Pencil` 或 `MessageSquare`）。
- **发言阶段**：
  - **当前发言人 = 本机**：**优先仅使用 §8.3 中部输入**；FAB **隐藏**或仅用于「查看规则/历史」次要入口，**禁止**与中部形成两个并列主输入口。
  - **非本人发言**：FAB 可打开 **历史描述列表** 或折叠面板（可选）；**不得**冒充发言输入。
- **投票阶段**：**推荐中部常驻投票区**；FAB 仅辅助或隐藏，与 `design.md` 一致。

### 8.5 按钮

- **主按钮**：金底深字或深底金边（保证对比度 ≥ WCAG AA）；hover 用 `--accent-bright`。
- **次按钮**：玻璃 + `border-white/15`。

### 8.6 描述输入 / 投票

- **发言输入**：以 **§8.3-B 中部输入态** 为主路径；抽屉/模态仅作 H5 窄屏 **可选等价布局**（须与中部 **同一套** 字段与校验，不能两套逻辑）。
- 投票（及发言倒计时若用环）**环形进度**：时长取自房间配置，**默认 30 秒**；金 → 琥珀警戒色（最后几秒可加强调，见 §7.4）。

### 8.7 模态 / 抽屉

- 背景：`bg-black/60 backdrop-blur-sm`
- 容器：`bg-[var(--bg-card)] border border-white/10 rounded-2xl`

- **`lg`（1024px）及以上**：启用 **左右双列玩家 + 中部舞台** 主布局。
- **以下**：折叠为 **顶栏 + 玩家横滑/Tab + 舞台 + FAB**，逻辑不变。

## 10. 无障碍与反馈

- 焦点环可见；投票错误 `aria-live`。
- 身份与投票结果不仅依赖颜色（配文案）。
- **音频**：等待态 BGM 与短音效须遵守 **浏览器自动播放策略**（必要时在用户首次点击/触摸后再 `play()`）；提供 **静音** 时不得仅隐藏图标而仍播放。
