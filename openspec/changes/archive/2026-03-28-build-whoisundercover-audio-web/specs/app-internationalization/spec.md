## ADDED Requirements

### Requirement: 品牌字标不参与翻译

界面顶栏等处的 **Logo 英文产品字标**必须固定显示为 **`Who Is Spy`**，且**不得**因当前界面语言为简体中文而改为「谁是卧底」或其他译法（与 [`docs/project-meta.md`](../../../../../docs/project-meta.md) 一致）。

#### Scenario: 中文界面下的顶栏

- **当** 用户处于 `zh` 语言路由
- **则** 顶栏 Logo 旁字标仍为 **`Who Is Spy`**

### Requirement: MVP 双语区域设置

系统必须在 MVP 阶段支持 **简体中文**与 **英文** 两种界面语言；不得在未更新文案文件的情况下在界面组件内硬编码面向用户的完整句子（**品牌字标 `Who Is Spy` 与调试日志除外**）。

#### Scenario: 用户切换为英文

- **当** 用户将界面语言切换为 English
- **则** 导航、按钮、空状态、错误提示、阶段标签等 UI 文案以英文展示

#### Scenario: 用户切换为简体中文

- **当** 用户将界面语言切换为简体中文
- **则** 上述 UI 文案以简体中文展示

### Requirement: 路由级语言前缀

Web 应用必须使用 **URL 语言段**区分语言（例如 `/zh/...` 与 `/en/...`），以便分享链接、刷新与 SEO 行为一致；**默认语言**为 **简体中文**（`zh`）。根路径 `/` 必须将用户导向默认语言的等价入口（重定向至 `/zh` 或框架约定的默认处理）。

#### Scenario: 访问带语言前缀的首页

- **当** 用户访问 `/zh` 或 `/en` 下的契约首页
- **则** 页面内容与该语言一致，且同一路由结构在两种语言下均可用

### Requirement: 文案集中管理

所有 MVP 可见字符串必须存放在仓库内 **集中文案文件**（如 `messages/zh.json`、`messages/en.json` 或按模块拆分但须双份同步），键名稳定、可检索；新增功能须同时补齐两种语言条目。

#### Scenario: 新增按钮文案

- **当** 开发新增需要展示的按钮标签
- **则** 在 `zh` 与 `en` 文案源中均存在对应键值，缺失时构建或 CI 可发现（若已配置检查）

### Requirement: 语言偏好持久化

客户端必须将用户选择的语言写入 **localStorage**（建议键 **`whoisspy_locale_v1`**，值为 `zh` 或 `en`），以便下次访问时优先匹配；若与 URL 语言段冲突，**以 URL 为准**或按产品约定在 `design.md` 中说明的统一策略执行（须在实现中写清并文档化）。

#### Scenario: 再次访问记住语言

- **当** 用户曾选择英文并保存偏好后关闭浏览器再打开站点
- **则** 进入应用时使用该偏好对应的语言路径或等价重定向

### Requirement: 游戏词库与 UI 文案分离

**系统词库**（`data/system-word-pairs.json`）中的词条为**游戏内容**，MVP **不要求**将词对翻译为英文；英文界面下仍可使用中文词对进行游戏，仅 **UI 壳层**英文化。若未来增加英文词库，须单独变更规格。

#### Scenario: 英文界面下进行游戏

- **当** 界面为英文且一局游戏发词
- **则** 平民与卧底收到的词语仍来自现有中文词库数据，不因界面语言而自动翻译
